# Документация

- `deploy.md` — развёртывание на Railway с нуля (аккаунт, проект, переменные, том).
- `bitrix-setup.md` — настройка Bitrix24: входящий вебхук, права, категории и статусы.
- `user.md` — памятка сотруднику: как отправлять заявки боту.
- `extending.md` — добавление категории, стадии, поля CRM или нового канала.

## Как запустить проект локально

```bash
python -m venv .venv
.venv/Scripts/activate   # Windows; на Linux: source .venv/bin/activate
pip install -r requirements-dev.txt
cp .env.example .env     # заполнить значения
python -m app.main
```

Тесты не требуют сети и токенов:

```bash
pytest tests/ -v
```

## Как устроен код

- `app/main.py` собирает бота: whitelist проверяет сообщения и callback-запросы,
  а логирование, лимит частоты и дедупликация применяются к сообщениям. Изоляция
  событий последовательно обрабатывает события одного чата и очищает
  неиспользуемые блокировки. Роутеры подключаются в фактическом порядке:
  `start → search → voice → messages → fallback`.
- `app/services/bitrix.py` — вся работа с CRM. Функции принимают клиент
  первым аргументом, поэтому тестируются без сети.
- Защита от дублей: ключ сообщения (у forward — по исходному сообщению)
  хранится в SQLite, а ключ `UF_CRM_TG_MSG_ID` в сделке страхует от дубля
  даже при потере локальной базы.
