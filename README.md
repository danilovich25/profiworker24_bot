# profiworker24_bot

Telegram-бот `@Profiworker24_manager_bot` принимает заявки сотрудников и
записывает контакты, сделки и задачи-напоминания в Bitrix24.

## Что умеет

- принимает заявки свободным текстом и голосовыми сообщениями;
- разбирает имя, телефон, организацию, адрес, категорию, описание, срок,
  доход и расход;
- задаёт уточняющие вопросы, а при недоступном автоматическом разборе собирает
  заявку пошагово;
- показывает карточку перед записью с действиями «Создать», «Изменить» и
  «Отмена»;
- находит контакт по телефону и использует его повторно, а при необходимости
  создаёт новый;
- создаёт сделку в Bitrix24 и возвращает её ID как номер заявки;
- хранит категорию услуги, доход, расход и рассчитанную прибыль в сделке;
- хранит организацию в контакте и позволяет искать по ней;
- создаёт задачи-напоминания Bitrix24 с распознанным дедлайном;
- ищет сделки по телефону, номеру заявки, имени и организации;
- показывает 10 последних заявок;
- защищает от повторной доставки сообщений, повторных нажатий и дублей после
  сетевых сбоев;
- ограничивает доступ списком Telegram ID сотрудников;
- при необходимости отправляет обезличенные ошибки в Sentry.

Старая версия бота на `telebot` сохранена в ветке `legacy/telebot-mvp`.

## Структура проекта

```text
app/
  main.py            # точка входа, сборка и запуск long polling
  config.py          # настройки из переменных окружения
  schemas.py         # структура заявки, категории и интенты
  db.py              # SQLite: дедупликация, черновики и защитные записи
  handlers/          # команды, текст, голос, поиск и кнопки карточки
  middlewares/       # доступ, логирование, лимит частоты и дедупликация
  services/
    bitrix.py        # контакты, сделки, поиск, стадии и поля CRM
    llm.py           # разбор свободного текста
    speech.py        # распознавание голосовых
    tasks.py         # задачи-напоминания
tests/               # офлайн-тесты без обращений к внешним API
docs/                # инструкции для владельца и сотрудников
```

## Запуск локально

Требуется Python 3.12.

```bash
python -m venv .venv
.venv/Scripts/activate   # Windows; на Linux: source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python -m app.main
```

Перед запуском заполните `.env` и оставьте для локальной базы
`DB_PATH=./bot.db`. Этот файл исключён из Git и не должен попадать в репозиторий.

## Тесты и проверка стиля

```bash
pip install -r requirements-dev.txt
pytest tests/ -q
ruff check app tests
```

Тесты используют подмены внешних клиентов и не требуют рабочих токенов или
доступа в интернет.

## Переменные окружения

| Переменная | Обязательность | Назначение |
|---|---|---|
| `BOT_TOKEN` | да | Токен Telegram-бота от @BotFather. |
| `BITRIX_WEBHOOK` | для работы с CRM | Базовый URL входящего вебхука Bitrix24. |
| `YC_API_KEY` | для разбора и голоса | API-ключ сервисного аккаунта Yandex Cloud. |
| `YC_FOLDER_ID` | для разбора и голоса | Идентификатор каталога Yandex Cloud. |
| `ALLOWED_TG_IDS` | да на проде | Telegram ID сотрудников через запятую, точку с запятой или пробел. Пустое значение закрывает доступ всем. |
| `ALLOW_ALL_USERS` | нет | Явно открыть доступ всем (`true`); только для закрытого стенда, на проде — `false`. |
| `DB_PATH` | да | Путь к SQLite: локально — `./bot.db`, на Railway — `/data/bot.db`. |
| `SENTRY_DSN` | нет | DSN проекта Sentry; пустое значение отключает интеграцию. |
| `TZ` | да | Часовой пояс для относительных дат и дедлайнов, рабочее значение — `Asia/Vladivostok`. |

Секреты хранятся только в переменных окружения. Токен Telegram, URL вебхука и
ключ Yandex Cloud нельзя добавлять в код, `.env.example` или историю Git.

## Используемые API

- **Telegram Bot API** — получение обновлений через long polling, команды и
  клавиатуры, отправка сообщений, получение голосовых файлов.
- **Bitrix24 REST API**:
  - `crm.contact.*` — поиск, создание и обновление контактов;
  - `crm.duplicate.findbycomm` — точный поиск контакта по телефону;
  - `crm.deal.*` — создание, чтение и поиск сделок;
  - `crm.status.list` — названия стадий для результатов поиска;
  - `crm.deal.userfield.*` и `crm.contact.userfield.*` — создание и проверка
    обязательных полей;
  - `crm.timeline.comment.add` — история повторного обращения у контакта;
  - `tasks.task.add` и `tasks.task.list` — создание и защита от дублей задач-
    напоминаний.
- **YandexGPT** через Yandex AI Studio SDK — структурированный разбор свободной
  фразы в поля заявки и определение напоминаний.
- **Yandex SpeechKit v1** — синхронное распознавание голосовых в формате OGG на русском
  языке.
